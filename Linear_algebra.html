What Is Matrices?

There are many distinct types of matrices, and operations on vectors and matrices play a significant role in linear algebra. When learning linear algebra, especially the portions applicable to machine learning, there are a few sorts of matrices that you might run into repeatedly.

The fundamental units of data science are matrices. They emerge across languages in a variety of avatars. From Python's numpy arrays to R's dataframes to MATLAB's matrices. In its simplest form, a matrix is simply a group of numbers arranged in a rectangular or array-like pattern. This could be an abstract structure, a network, or even an image.

The matrix, or matrices as the plural, is surprisingly more prevalent than you may imagine. All of our Photoshop-edited photographs use matrices to render visuals by processing linear transformations. A geometric object's linear transformation can be represented by a "Square matrix."


For example, the reflection of an item in the vertical Y-axis is produced using the matrix in the Cartesian X-Y plane. This would produce an assassin's upside-down reflection in a pool of blood in a video game. The matrix would be more challenging to use to extend or contract the reflection if the video game featured curved reflecting surfaces, like a room full of mirrors.

Matrix analysis is used in applied physics to examine optical, quantum, and electrical circuits. Engineering fields like chemical and aerospace all call for precisely calibrated calculations that are produced via matrix transformations. In hospitals, matrices are used to produce the outputs of medical imaging procedures like CAT scans and MRIs. In programming, messages are coded and encrypted using matrices and inverse matrices. Matrix-based motions for robots are the fundamental building blocks of robotics and automation. Matrix calculations are used to acquire the inputs for controlling robots.

Basic operations using matrices

Matrices just like numbers can be added, subtracted and multiplied. The division though is slightly nuanced. Not all matrices can be divided. There are certain rules for even the addition, subtraction and multiplication.

1. Matrices Addition and Subtraction

Matrix addition is quite similar to simple addition in mathematics. All you require is that the matrices be added in the same sequence. When you add matrices on your own, this concept will become clear. A matrix C (m*n) is created by adding two matrices A (m*n) and B (m*n). The components of C consist of the sum of the corresponding components of A and B.

Subtraction works similarly,


"The important thing to remember is that only matrices with the same number of rows and columns, or the same order (*order = rows x columns), can be added to or subtracted from."

  • The ratio of the number of rows of A to those of B
  • The ratio of the number of columns in column A to column B

Points to note :

  • Matrix addition is commutative, meaning that A + B = B + A.
  • Matrix addition is associative, meaning that A + (B + C) = (A + B) + C
  • Matrix subtraction is non-commutative, which indicates that A - B = B - A.
  • Matrix subtraction is non-associative, i.e., A - (B - C) = (A - B) - C
  • If A and B are ordered differently, A + B and A - B cannot be computed.
  • The complexity of an addition or subtraction operation is O(m^n), where m^n is the order of the matrices.

However, multiplication has a little complexity.

2. Matrix Multiplication

The multiplication of two matrices A(m*n) and B(n*p) gives a matrix C(m*p). Notice that for multiplication you do not need the rows/columns of A and B to be the same. You only need

  • No. of Columns of A = No. of Rows of B, or
  • No. of Columns of B = No. of Rows of A.

To calculate the top-left element of the resulting matrix C, multiply elements of 1st row of A with 1st column of B and add them.

Points to note :

  • Matrix multiplication is non-commutative, therefore A * B ≠ B * A
  • Matrix multiplication is associative, therefore A * (B^C) = (A^B) * C
  • The existence of A^B does not prove that B^A exists.
  • The complexity of the operation (A^B) is O(m*n*p), where m^n and n*p are the respective orders of A and B.
  • Matrix C has an order of m^p when calculated as A^B, where m^n and n^p represent the order of A and B, respectively.

Identity Matrix

We just mentioned the "Identity Matrix". It is the matrix equivalent of the number "1":

  • It is "Square" (has same number of rows as columns),
  • It has '1's on the diagonal and '0's everywhere else.
  • Its symbol is the capital letter I.

The Identity Matrix can be 2×2 in size, or 3×3, 4×4, etc...

Eigenvalues and Eigenvectors

An eigenvector does not change direction in a transformation is a straightforward illustration.



How do we find that vector? Eigen is a German term that signifies "own" or "innate," as in being a part of the parent matrix. It is pronounced "eye-gan". Since eigenvectors are unit vectors, their length or magnitude is always equal to one. Right vectors, which simply means a column vector, are a common name for them. The length or magnitude of the eigenvectors is determined by the eigenvalues, which are coefficients applied to the eigenvectors.

Eigenvalues in Principal Component Analysis

There is a lot of data involved in machine learning. Programmers will have a field day if there is any way to shrink the data without sacrificing the accurate representation of those values.
PCA, often known as principal component analysis, is used to reduce dimensionality. Finding meaningful features becomes more challenging as datasets grow larger. In order to make the machine learning model more robust, it is therefore necessary to determine whether two variables are correlated with one another.

Let's talk about PCA and take a quick look at how it works:

  • The data must first be centered, which entails removing the average or mean from each attribute. It is now simpler to compute the covariance matrix.
  • The degree to which two variables are dependent or independent of one another is indicated by their covariance.
  • If Y grows with an increase in X and drops with a reduction in X, two variables, say X and Y, are said to be in positive correlation.

Any vector in the feature space can be multiplied by this covariance matrix to produce a vector, the slope of which on repeated multiplications will gradually converge towards the maximum variance. As a result, the dimension with the highest variance in the data is indicated by this multiplication of a random vector with a covariance matrix.
The need to locate these convergent spots in the data becomes critical. One method is to multiply the covariance matrix with the vectors to see which ones do not turn on.
As a result, the vector's range widens without turning. The span is a scalar quantity, and for a given value, there is a vector that doesn't change in a particular way. This perfectly fits the underlying premise of Eigenvectors.
